The MCP server exposes a `field-types` resource at `opnform://reference/field-types`. AI agents can read this resource to discover all available field types, their configurable options, and examples before calling `create-form`.

<Tip>
Most MCP clients automatically read available resources. If your agent doesn't, you can ask it to "read the field-types resource" to load the catalog.
</Tip>

## Layout blocks

Layout blocks are non-input elements that control form presentation.

| Type | Title | Description |
| ---- | ----- | ----------- |
| `nf-text` | Text Block | Static text/content block (not an input) |
| `nf-page-break` | Page Break | Splits form into multiple pages (classic mode only) |
| `nf-divider` | Divider | Horizontal divider line (classic mode only) |
| `nf-image` | Image | Image block (classic mode only) |

## Common field properties

Every field (both input and layout) supports these properties:

| Property | Type | Default | Description |
| -------- | ---- | ------- | ----------- |
| `name` | string | *required* | Field label shown to form respondents |
| `type` | string | *required* | One of the types listed above |
| `required` | boolean | `false` | Whether the field is required |
| `help` | string | â€” | Help text shown below the field |
| `hidden` | boolean | `false` | Hide the field from the form |
| `placeholder` | string | â€” | Placeholder text (where applicable) |

<Note>
Field `id` values are auto-generated as UUIDs when omitted. You can provide your own IDs if you need stable references.
</Note>
