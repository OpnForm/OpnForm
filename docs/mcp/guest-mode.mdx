---
title: "Guest Mode"
description: "Let AI agents draft forms without authentication and save them after registration."
---

The guest MCP endpoint lets users try OpnForm through an AI agent without creating an account first. The agent drafts a form, guides the user through registration, then persists the form with a single tool call.

## Guest endpoint

The guest server is available at `/mcp/guest` with no authentication required:

```json mcp.json
{
  "mcpServers": {
    "opnform-guest": {
      "url": "https://api.opnform.com/mcp/guest"
    }
  }
}
```

<Note>
Self-hosted instances use your own API domain: `https://your-api-domain.com/mcp/guest`
</Note>

## Available tools

The guest endpoint exposes a limited set:

| Tool / Resource | Description |
| --------------- | ----------- |
| `draft-form` | Build a form draft — returns JSON, nothing is persisted |
| `field-types` resource | Complete catalog of available field types |

All other tools (`create-form`, `list-submissions`, etc.) require the authenticated `/mcp` endpoint.

## How the flow works

<Steps>
<Step title="Agent drafts the form">

The user describes what they want ("create a feedback form with rating and comments"), and the agent calls `draft-form` with the appropriate fields. The tool returns validated form JSON, a registration URL, and next-step instructions.

```json draft-form response
{
  "form_data": {
    "title": "Feedback Form",
    "visibility": "draft",
    "properties": [
      { "id": "...", "type": "rating", "name": "How would you rate us?" },
      { "id": "...", "type": "text", "name": "Comments", "multi_lines": true }
    ],
    "theme": "default"
  },
  "register_url": "https://opnform.com/register",
  "next_steps": "1. Tell the user: \"Your form is ready! Open this link to create your OpnForm account: https://opnform.com/register\" ..."
}
```

<Info>
The `form_data` object is held in the agent's context — nothing is stored on the server.
</Info>
</Step>

<Step title="User registers">

The agent presents the registration URL to the user. The user opens it in their browser, creates an account, and generates an API token in **Settings → Access Tokens**.
</Step>

<Step title="User provides the token">

The user gives the API token back to the agent. The agent reconfigures the MCP connection to use the authenticated `/mcp` endpoint with the Bearer token.
</Step>

<Step title="Agent saves the form">

The agent calls `list-workspaces` to get the user's workspace ID, then calls `create-form` with the `form_data` from step 1 and the `workspace_id`. The form is persisted and the agent returns the share URL.

<Check>
The form is now live and accessible at its share URL.
</Check>
</Step>
</Steps>

## Combining guest and authenticated servers

You can configure both endpoints simultaneously. The AI client will use the guest server when no token is available and switch to the authenticated server once the user provides one:

```json mcp.json
{
  "mcpServers": {
    "opnform-guest": {
      "url": "https://api.opnform.com/mcp/guest"
    },
    "opnform": {
      "url": "https://api.opnform.com/mcp",
      "headers": {
        "Authorization": "Bearer YOUR_API_TOKEN"
      }
    }
  }
}
```

<Warning>
The `draft-form` tool is also available on the authenticated endpoint. If the user is already authenticated, the agent can call `create-form` directly instead — no need for the draft-then-save flow.
</Warning>
